Lagring — records, blokker, buffer

Bratsbergs notater kap 1–8. Hvordan en rad faktisk er bytes på en disk.

01 · DBMS-arkitektur

Komponentene

[Stubber:]

  • SQL-kompilator + katalog → algebra-tre
  • Optimizer → plan med kostnad
  • Executor → kjører plan via buffer/lock/log
  • JIT-kompilering i moderne DBMS-er
02 · Device og blokker

Disk er enheten

  • HDD (rotating) vs. SSD (flash) — forskjellig layout
  • Raw partition vs. fil i filsystem
  • BlockId = (deviceId, offset)
  • 16 KiB er typisk blokkstørrelse
03 · Record-format

Tre måter å pakke en rad

  • Fixed length — alle attributter på fast offset
  • Record vector — pekere til hvert felt (variabel lengde)
  • Delimiter — spesielle tegn som markerer slutt på felt/record
  • Row store vs. column store
04 · Block-format

Slot directory

  • Records vokser fra start, slot directory fra slutt
  • Page directory sortert på key — letter binærsøk
  • RID = (BlockId, slot) — semilogisk
  • Flip/flop — atomisk read/write-stempel
  • Header: identifikator, recovery, checksum
05 · Database-buffer

Hold ting i RAM

  • Buffer = stor andel av main memory
  • Hash-indeks fra BlockId til buffer-frame
  • Pinning — beskytter mot eviction
  • Erstatningsstrategier: LRU, MRU, clock
  • Skitne blokker — må skrives før eviction
06 · Heapfiler

Usortert lagring

  • Records lagres i tilfeldig orden
  • Insert er billig, søk er dyrt (full scan)
  • Free space management (FSM)
  • Page splits når blokk er full
07 · Hashing

Konstant-tid oppslag på likhet

  • Statisk hashing — kollisjoner og bucket overflow
  • Dynamisk hashing (extendible, linear)
  • Begrensninger: ingen range-spørringer
