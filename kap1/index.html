Helhetsbilde

Hva er en database

Bullet-stubber:

  • Persistente, strukturerte data som mange klienter deler
  • Hvorfor DBMS over filsystem (samtidighet, integritet, recovery, abstraksjon)
  • Datamodeller: relasjonell vs. dokument vs. graf
  • Skjema vs. instans
Arkitektur

DBMS-en i fugleperspektiv

Foregriper Del 2 — alle delene som dukker opp senere.

  • SQL-kompilator → algebra-tre
  • Optimizer → kostnadsbasert plan
  • Executor → kjører plan, snakker med buffer/lås/log
  • Buffer pool → blokker i minnet
  • Disk → heapfiler + indekser
  • Logg → garanterer durability + atomicity
ACID

Fire bokstaver, fire garantier

  • Atomicity — alt-eller-ingenting
  • Consistency — bevarer integritetskrav
  • Isolation — som om transaksjonen var alene
  • Durability — etter commit overlever data krasj
