01 · Aksessmetoder

Tre måter å hente rader

[Stubber:]

  • Full table scan — alle blokker
  • Index scan — gå gjennom indeks, slå opp i tabell
  • Index-only scan — alle nødvendige kolonner i indeksen
  • Bitmap scan — kombinerer flere indekser
02 · External merge sort

Sortering når data ikke får plass i RAM

  • Pass 1: les inn B blokker, sorter internt, skriv ut "runs"
  • Pass 2+: merge B−1 runs om gangen
  • Antall passes: ⌈log_(B−1)(N/B)⌉
  • I/O-kost: 2N · (antall passes)
03 · Nested loop join

For hver rad i ytre … for hver rad i indre …

  • Naiv: O(|R| · |S|) blokker
  • Block nested loop — hent en blokk om gangen
  • Index nested loop — bruk indeks på indre
  • Når lønner det seg: liten ytre tabell, indeks på indre
04 · Sort-merge join

Sorter begge, gå parallelt

  • Sorter R og S på join-attributt
  • Gå parallelt med to pekere
  • Bra hvis input allerede er sortert
05 · Hash join

Build + probe

  • Build phase: lag hash-tabell på mindre tabell
  • Probe phase: for hver rad i andre tabell, slå opp
  • Grace hash join når ikke alt får plass i minnet
06 · Kostnadsestimering

Statistikk og selektivitet

  • Tellinger, distinct values, histogrammer
  • Selektivitet av predikater
  • Optimalisator velger plan med lavest forventet I/O-kost
07 · EXPLAIN

Be DBMS om planen

  • EXPLAIN / EXPLAIN ANALYZE
  • Lese plantreet: rotnoden er resultatet
  • Kostnad i pseudo-units, faktisk tid med ANALYZE
